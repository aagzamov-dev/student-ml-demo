# Project Guide

## What This Project Is

This project demonstrates one machine learning task:

**Binary classification**

The model answers one question:

**Will this student pass the final exam?**

This is better for presentation because it is simple, easy to explain, and easy to test live.

## How The Data Is Created

The dataset is not collected from a real school. It is **synthetic data** generated by code in [scripts/generate_data.py](/mnt/d/KING_FILES/Projects/student-ml-demo/scripts/generate_data.py).

The generator creates student features such as:

- `study_hours_per_day`
- `attendance_percent`
- `assignments_completed`
- `sleep_hours`
- `previous_score`
- `internet_usage_hours`
- `participation_score`
- `extra_tutoring`
- `practice_tests_completed`
- `stress_level`

Then it calculates:

- `final_exam_score`
- `pass_fail`

How it works:

1. The script randomly creates realistic student habits and scores.
2. It builds a final exam score using a weighted formula.
3. Students with score `>= 50` become `pass_fail = 1`.
4. Students with score `< 50` become `pass_fail = 0`.

Output file:

- `data/students.csv`

## What The Model Is Training

Training happens in [scripts/train_models.py](/mnt/d/KING_FILES/Projects/student-ml-demo/scripts/train_models.py).

The project trains one model only:

- `RandomForestClassifier`

Input:

- the 10 student behavior features

Target:

- `pass_fail`

Training flow:

1. Load the dataset from `data/students.csv`
2. Validate that the required columns exist
3. Scale the input features with `StandardScaler`
4. Split data into training and testing sets
5. Train the classifier
6. Measure model quality
7. Save model files

Saved files:

- `models/classifier.pkl`
- `models/scaler.pkl`
- `models/metrics.json`

## What Metrics Are Calculated

The project saves these metrics:

- `accuracy`
- `precision`
- `recall`
- `f1_score`
- confusion matrix values

Why they matter:

- `accuracy`: overall correct predictions
- `precision`: when model says pass, how often it is right
- `recall`: how well the model finds actual passing students
- `f1_score`: balance between precision and recall

## What The Test Script Does

Testing is in [scripts/test_pipeline.py](/mnt/d/KING_FILES/Projects/student-ml-demo/scripts/test_pipeline.py).

It checks:

1. Required files exist
2. Model and scaler load correctly
3. Dataset and metrics load correctly
4. A sample student can be predicted successfully
5. The output probability is in a valid range

This is not a unit test framework. It is a **pipeline smoke test**.

That means:

- it quickly verifies the project is working end to end
- it checks the most important parts of the workflow

## How Prediction Works

Prediction logic is in [src/predict.py](/mnt/d/KING_FILES/Projects/student-ml-demo/src/predict.py).

When you enter student data:

1. Inputs are converted into a DataFrame
2. Inputs are scaled using the saved scaler
3. The classifier calculates pass probability
4. The app shows:
   - `Likely Pass` or `Likely Fail`
   - pass probability
   - fail risk

## How The UI Works

The Streamlit app is in [app.py](/mnt/d/KING_FILES/Projects/student-ml-demo/app.py).

It has 4 pages.

### Overview

Purpose:

- quick project summary
- show dataset size
- show pass rate
- show distribution charts

### Test Student

Purpose:

- live demo page
- test different student profiles

Input methods:

- `Quick scenario`
- `Dataset student`
- `Custom editable row`

This page lets you:

- choose a ready-made example
- load a real row from the dataset
- edit one custom student row directly
- move sliders to see prediction change instantly

### Student Data

Purpose:

- show all generated students
- inspect individual rows
- load any student into the test page

### Model Evidence

Purpose:

- show model quality
- show confusion matrix
- show feature importance

This is useful if your teacher asks:

- how well does it work?
- what affects the prediction most?

## Why `.env` Exists

The file [.env](/mnt/d/KING_FILES/Projects/student-ml-demo/.env) is used to reduce a common Windows warning from `joblib/loky`.

It contains:

```text
LOKY_MAX_CPU_COUNT=4
```

This helps avoid noisy CPU core warnings when training.
